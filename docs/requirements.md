# 💎 쥬얼리 쇼핑몰 요구사항 정의서

## 1. 프로젝트 개요

- **목적:** TypeORM 및 NestJS 기반의 확장 가능한 쥬얼리 이커머스 백엔드 구축
- **핵심 가치:** 쥬얼리 특유의 복잡한 옵션(반지 호수, 재질, 각인 등) 처리 및 트랜잭션 안정성 확보

## 2. 핵심 도메인 요구사항

### 2.1. 회원 (User)

- [ ] 이메일 기반의 회원가입 및 로그인을 지원한다.
- [ ] 권한(Role) 관리를 통해 일반 사용자와 관리자를 구분한다.

### 2.2. 상품 (Product)

- [ ] 상품의 기본 정보(이름, 가격, 설명)를 관리한다.
- [ ] 쥬얼리의 다양한 옵션(금 색상, 반지 호수, 보석 종류 등)을 유연하게 저장하고 조회할 수 있어야 한다. (JSON 활용)
- [ ] 재고(Stock) 관리가 정확하게 이루어져야 한다.

### 2.3. 주문 및 결제 (Order)

- [ ] 장바구니에 담긴 상품들을 결제하여 주문을 생성한다.
- [ ] 주문 상태(결제대기 -> 결제완료 -> 제작중 -> 배송중 -> 배송완료)를 추적할 수 있어야 한다.
- [ ] 결제 완료 시 해당 상품의 재고가 안전하게 차감되어야 한다. (트랜잭션 보장)
